# Product

## Register

product

## Users

Margonem MMORPG guild members, primarily Polish-speaking players. They use Tepirek to coordinate guild activities: managing events, running auctions, tracking clan skills, and keeping tabs on member rosters. Context is a gaming environment where efficiency matters but the interface should still feel like it belongs to the game world, not a detached corporate tool.

## Product Purpose

Replace spreadsheets and fragmented coordination with a unified guild management platform. Success means guild leaders and members can schedule events, manage bets, run item auctions, track skills across ranges, and verify player status without friction. The tool should fade into the background so the game stays center stage.

## Brand Personality

Modern, minimalistic, game-vibe. The interface carries the soul of Margonem without being a literal reskin: clean lines, purposeful density, and subtle nods to RPG interface language (structured data, clear hierarchies, status indicators). Voice is direct and functional; Polish copy should feel natural, not translated.

## Anti-references

- Generic SaaS dashboard clichés: identical card grids, hero-metric templates, gradient text accents, side-stripe borders.
- Cluttered MMO fan-sites with neon overload and information chaos.
- Overly corporate / sterile enterprise tools that strip away all personality and context.

## Design Principles

1. **Game world, not game skin.** The UI should feel native to the Margonem universe without relying on gimmicky themed decorations. Subtle color direction and structural references to RPG interfaces are enough.
2. **Efficiency first.** Every screen optimizes for its primary task. No decorative friction, no restated headings, no intros that repeat the title.
3. **Clarity over clutter.** Old-school MMO interfaces are information-dense by default. Resist that impulse: present data with clear hierarchy, generous whitespace, and focused layouts.
4. **One tool, many workflows.** Events, auctions, skills, tasks, and calculators each deserve tailored layouts rather than being forced into a single repeated card or table pattern.
5. **Polish through restraint.** Modern and minimal means intentional color, precise spacing, and no visual noise. Every element earns its place.

## Accessibility & Inclusion

- WCAG 2.1 AA compliance.
- Keyboard navigation support for all interactive elements.
- Screen-reader friendly data tables and forms.
- Respect `prefers-reduced-motion` for any animated transitions.
